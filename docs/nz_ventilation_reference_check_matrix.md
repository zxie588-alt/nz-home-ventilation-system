# NZ Ventilation Reference Check Matrix

This matrix records the local New Zealand references used in the project. It is a preliminary design-context check for portfolio evidence. It is not a building consent submission, producer statement, compliance certificate or professional engineering statement.

## Reference Matrix

| Reference | Why it matters | Project response | Evidence file | Status |
| --- | --- | --- | --- | --- |
| NZ Building Code G4 - Ventilation | Establishes the ventilation design context for occupied spaces, outdoor air and extract ventilation. | The concept uses balanced MVHR/ERV supply and extract with dedicated kitchen, bathroom and laundry extract points. | `docs/nz_ventilation_preliminary_design_note.md` | Preliminary reference check |
| NZBC G4/AS1 fifth edition, effective 28 July 2025 | Provides an acceptable-solution pathway and current terminology for ventilation review. | Used as a reference context only; no consent claim is made. | `drawings/nz_ventilation_m101_design_review_sheet.svg` | Context only |
| NZ Building Code H1 - Energy Efficiency | Ventilation and heat recovery affect energy use and heating/cooling loads. | MVHR/ERV concept includes heat-recovery sense check and fan-energy discussion. | `docs/nz_ventilation_fan_energy_noise_check.md` | Preliminary energy check |
| Healthy Homes ventilation standard | Provides practical residential checks for kitchens, bathrooms and openable ventilation. | Kitchen and bathroom boost targets are separated from normal balanced ventilation. | `docs/nz_ventilation_airflow_schedule.csv` | Point-check context |
| Mitsubishi Electric Lossnay LGH-RVX3-E data | Needed for a manufacturer-based unit-size check. | LGH-35RVX3-E selected because normal and boost targets sit within its published range with headroom. | `docs/nz_ventilation_equipment_selection.md` | Concept selection |
| Project commissioning plan | A design should be measurable and testable. | Test cases cover airflow balance, CO2 response, RH response, pressure trend, fan power and data export. | `docs/nz_ventilation_commissioning_test_plan.md` | Portfolio validation pathway |

## Boundaries

- The project uses a concept dwelling and simplified duct geometry.
- It does not include a real architectural plan, window schedule, final diffuser/grille data, full acoustic design or professional compliance review.
- The wording should remain "reference check", "preliminary check" or "design-context check", not "code compliant".

## Strong Interview Framing

> I treated NZ regulations as design context rather than a claim of compliance. I mapped G4, H1, Healthy Homes and manufacturer data to specific project evidence: airflow schedule, equipment selection, commissioning plan, dashboard logic and design limitations.
